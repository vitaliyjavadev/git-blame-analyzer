# Git Blame Analyzer

[![GitHub license](https://img.shields.io/badge/license-MIT-blue.svg)](https://mit-license.org/)
[![Bash](https://img.shields.io/badge/bash-5.0+-green.svg)](https://www.gnu.org/software/bash/)
[![Git](https://img.shields.io/badge/git-2.0+-blue.svg)](https://git-scm.com/)

Инструмент для анализа распределения авторства строк кода в Git-репозитории по поддерживаемым расширениям файлов.

## Статус

**Стабильная версия:** v1.0  
**Язык:** Bash  
**Совместимость:** Linux, macOS, Windows (WSL/Cygwin)

## Установка

1. Склонируйте репозиторий или скачайте файл [`git-blame-analyzer.sh`](git-blame-analyzer.sh)
2. Сделайте файл исполняемым:

```bash
chmod +x git-blame-analyzer.sh
```

3. Убедитесь, что у вас установлены необходимые зависимости:

```bash
git --version
bash --version
```

## Использование

Запустите скрипт из корня Git-репозитория или укажите путь к нему:

```bash
./git-blame-analyzer.sh [путь_к_репозиторию]
```

Если путь не указан, анализ выполняется для текущей директории.

## Конфигурация

### Поддерживаемые расширения файлов

Расширения файлов для анализа определяются в строке 17 [`git-blame-analyzer.sh`](git-blame-analyzer.sh) в регулярном выражении: ```bash
grep -E '\.(kt|sql|ya?mlf|xml|properties)$'
```

**Текущий список:**
- Kotlin: `.kt`, `.kts`
- Java: `.java` (закомментировано в строке 10)
- SQL: `.sql`
- YAML: `.yml`, `.yaml`
- YAML with Anchors: `.ya?l` (закомментировано в строке 10)
- XML: `.xml`
- Properties: `.properties`

**Изменение списка:** раскомментируйте нужные расширения в строке 10 или добавьте новые в регулярное выражение.

### Исключения из анализа

Скрипт использует команду `git ls-files --cached --others --exclude-standard`, которая автоматически учитывает:
- `.gitignore` в корне репозитория и поддиректориях
- `.git/info/exclude` (локальные исключения)
- `core.excludesFile` (глобальные исключения)

Файлы, соответствующие шаблонам в этих файлах, автоматически исключаются из анализа.

### Состояние Git-файлов

Скрипт анализирует файлы из следующих состояний Git:
- `--cached`: проиндексированные (staged) файлы
- `--others`: непроиндексированные (unstaged) файлы
- `--exclude-standard`: исключает файлы из `.gitignore`, `.git/info/exclude` и `core.excludesFile`

Таким образом, анализ охватывает **все файлы в рабочей директории**, которые отслеживаются или не отслеживаются Git, но исключает файлы, явно объявленные как игнорируемые.

## Аргументы командной строки

| Аргумент | Описание |
|----------|----------|
| `путь_к_репозиторию` | Опциональный путь к анализируемому Git-репозиторию (по умолчанию: текущая директория `.`) |

## Примеры

### Анализ текущего репозитория

```bash
./git-blame-analyzer.sh
```

### Анализ удаленного репозитория

```bash
./git-blame-analyzer.sh /path/to/project
```

### Анализ в пайплайне CI/CD

```bash
./git-blame-analyzer.sh . > blame_report.txt 2>&1
```

## Требования к окружению

- Bash 5.0+
- Git 2.0+
- Утилиты: `wc`, `sort`, `uniq`, `head`
- Доступ к Git-репозиторию (текущая директория или указанный путь)
